## Component Responsibilities

### 1. Jupyter Notebook (Frontend)
- **What**: User interface for writing and running code
- **Runs on**: Port 8888
- **Tech**: JupyterLab with custom extensions
- **Responsibilities**:
  - Display notebooks
  - Provide GPU selection UI
  - Execute code cells
  - Show results

### 2. Backend API (Middleware)
- **What**: Connects Jupyter to FlexAI platform
- **Runs on**: Port 8000
- **Tech**: FastAPI (Python)
- **Responsibilities**:
  - Handle GPU selection requests
  - Manage user sessions
  - Communicate with FlexAI API
  - Track usage and billing

### 3. Mock FlexAI API (Simulator)
- **What**: Simulates real FlexAI platform
- **Runs on**: Port 9000
- **Tech**: FastAPI (Python)
- **Responsibilities**:
  - Return mock GPU types
  - Simulate instance provisioning
  - Track mock instances
  - Respond instantly (no real waiting)

### 4. PostgreSQL (Database)
- **What**: Persistent data storage
- **Runs on**: Port 5432
- **Tech**: PostgreSQL 16
- **Stores**:
  - User sessions
  - Compute instances
  - Usage logs
  - Metadata

### 5. Redis (Cache)
- **What**: Fast in-memory cache
- **Runs on**: Port 6379
- **Tech**: Redis 7
- **Caches**:
  - Active sessions
  - GPU availability
  - Recent queries
  - Temporary data

## API Endpoints

### Backend API (localhost:8000)

```
GET  /health                    - Health check
GET  /api/compute/gpu-types     - List available GPUs
POST /api/compute/select        - Select and provision GPU
GET  /api/sessions/{id}         - Get session info
POST /api/sessions/stop         - Stop session
```

### Mock FlexAI API (localhost:9000)

```
GET  /health                           - Health check
GET  /v1/compute/gpu-types             - List mock GPUs
POST /v1/compute/instances             - Create instance
GET  /v1/compute/instances/{id}        - Get instance
POST /v1/compute/instances/{id}/stop   - Stop instance
DEL  /v1/compute/instances/{id}        - Delete instance
```
